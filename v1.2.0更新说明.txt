【新功能详解】

1. ✨ 双视图模式
   ────────────────────────────────────────────
   输出区域现在有两个选项卡：
   
   📄 原始输出
   - 保留传统的文本输出
   - 适合查看完整日志
   - 可复制粘贴
   
   📊 结构化视图 ⭐ 新增
   - 自动解析输出内容
   - 以表格形式展示
   - 可排序、筛选、统计
   - 一目了然，方便分析

2. 🔍 智能搜索筛选
   ────────────────────────────────────────────
   • 实时搜索：输入关键词立即过滤
   • 全字段匹配：搜索所有列的内容
   • 一键清除：快速重置筛选
   
   使用场景：
   - 在DLL列表中快速找到特定文件
   - 筛选特定类型的句柄
   - 查找某个版本的文件

3. 📈 自动统计信息
   ────────────────────────────────────────────
   • 显示记录总数
   • 执行后自动切换到结构化视图
   • 蓝色高亮显示统计信息

4. 📋 增强的复制功能
   ────────────────────────────────────────────
   • 复制原始输出（全部内容）
   • 复制选中行（格式化显示）
   • 复制到剪贴板（快捷操作）

5. 💾 导出CSV功能 ⭐
   ────────────────────────────────────────────
   • 将结构化数据导出为CSV
   • 支持Excel直接打开
   • UTF-8编码，支持中文
   • 自动生成时间戳文件名
   
   用途：
   - 深度分析（Excel/Python）
   - 生成报告
   - 数据存档

6. 🔄 列排序功能
   ────────────────────────────────────────────
   • 点击列标题即可排序
   • 升序排列
   • 适合查找最大/最小值

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【工具专属解析】

📌 ListDLLs 结构化视图
   列：DLL路径 | 版本 | 文件版本 | 发布者 | 描述 | 产品 | 创建时间
   
   特点：
   - 自动提取所有DLL的详细信息
   - 版本信息清晰展示
   - 方便统计DLL数量
   - 快速查找特定厂商的DLL

📌 Handle 结构化视图
   列：进程 | PID | 句柄 | 类型 | 路径或名称
   
   特点：
   - 每个句柄单独一行
   - 按类型筛选（File, Event, Section等）
   - 快速定位文件句柄
   - 统计句柄总数

📌 ProcDump
   保持原有文本输出（转储进度不需要结构化）

【使用示例】

场景1：统计进程加载了多少个DLL
────────────────────────────────────────────
1. 在进程列表选择目标进程
2. 右键 → 应用到 ListDLLs
3. 勾选"显示版本信息"
4. 点击"执行"
5. 自动切换到"结构化视图"
6. 顶部显示：共 XX 条记录 ✅
7. 可导出CSV进行深度分析

场景2：查找某个DLL的版本
────────────────────────────────────────────
1. 执行ListDLLs后切换到结构化视图
2. 在筛选框输入DLL名称（如"kernel32"）
3. 表格自动过滤，只显示匹配项
4. 查看版本列 ✅

场景3：查看进程打开了哪些文件
────────────────────────────────────────────
1. 执行Handle工具
2. 切换到结构化视图
3. 在筛选框输入"File"
4. 只显示文件类型的句柄 ✅
5. 点击"类型"列标题排序

场景4：导出数据用Excel分析
────────────────────────────────────────────
1. 执行任意工具（ListDLLs/Handle）
2. 切换到结构化视图
3. 点击"导出CSV"按钮
4. 选择保存位置
5. 用Excel打开查看 ✅
